# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？
### リモートリポジトリー
  専用のサーバに配置して複数人で共有するためのリポジトリ

### ローカルリポジトリ
  ユーザ一人ひとりが利用するために<bir>自分の手元のマシン上に配置するリポジトリ


## プッシュとマージの違いは何でしょうか？
### プッシュ
  ローカルリポジトリの変更をリモートリポジトリに送信する操作
### マージ
  分岐したブランチの履歴を統合する操作

## コミットとプッシュの違い
### コミット
  ローカルリポジトリの変更を記録する操作

### プッシュ
  ローカルリポジトリのコミットをリモートリポジトリにアップロードする操作

## コミットのメッセージはどのように書いてあげるのが最適でしょうか？
  50文字以内を目安に変更内容が一目でわかるように何が変更されたかだけでなく<biv>なぜその変更が必要だったのかを明確に伝える

## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？
  ローカルでマージは、自分のPCで直接変更を統合する作業
  プルリクエストマージは、リモートリポジトリ（GitHubなど）で変更のレビューを依頼し、承認後に統合する

## コンフリクトを起こしてしまった場合、どう対処すべきですか？
### リバース
  ブランチ元のポイントを変える

### マージ
  マージ先のブランチを違う方向にマージして解消する
